Getting Started

UI components can be used via CDN script or NPM library. Below we list instructions and examples on how to use the UI components.

Using Via CDN

  1. Add the following script tags to your html file. The version should match the types npm package you installed.

    2. <script src = "https://unpkg.com/@laserfiche/lf-ui-components@NPM_VERSION/cdn/laserfiche-ui-components.js" defer ></script>

    Choose a theme to style your components. Import the style sheet from the CDN by adding this line to your html file inside your application:

    <link href = "https://unpkg.com/@laserfiche/lf-ui-components@NPM_VERSION/cdn/lf-laserfiche-lite.css" rel = "stylesheet" />

    If you are not using Angular Material, you will also need to add some Angular Material styles:

    <link href = "https://unpkg.com/@laserfiche/lf-ui-components@NPM_VERSION/cdn/indigo-pink.css" rel = "stylesheet" />

  2. Install the types definition for the custom elements.

    npm install @laserfiche/types-lf-ui-components@NPM_VERSION

  3. Use the custom elements as defined in the documentation (lf-tree, etc.)
  4. There are other libraries available for use as well, that may be useful for you.
    • @laserfiche/lf-js-utils - this library contains utility functions that help avoid duplicate code.
    • @laserfiche/laserfiche-ui-components-services - this library contains implementations of the services used by the UI components. (Ex/ LfTreeService)
      1. These services connect with the Laserfiche Repository API to render real data from Laserfiche.
  5. Some components, specifically those with popups like dialogs, dropdowns, or menus, cannot be used inside jQuery modals due to our use of Angular Material. In order to use these components in a modal, we recommend using basic modals built with html/css/bootstrap.

Example: lf-checklist In an Angular Project Via CDN

In the example below, the implementation comes from the CDN and the types is provided in the @laserfiche/types-lf-ui-components NPM package.

  1. Follow through step 1 and 2 above.

  2. Inside ./app.module.ts, add CUSTOM_ELEMENTS_SCHEMA to @NgModule.schemas. This enables custom elements in your Angular application. 

      import {  NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';
  @NgModule({
    declarations: [
      AppComponent,
      ...
    ],
    imports: [
      ...
    ],
    schemas: [CUSTOM_ELEMENTS_SCHEMA]
  })

  3. In a html file,

    <lf-checklist #myChecklist (checklistChanged)="onChecklistChangedAsync($event)"></lf-checklist>
<!-- checklistChanged is a HTML custom event -->

    To add a handle for this element, in the .ts file,

    import { AfterViewInit, Component, ViewChild, ElementRef } from '@angular/core';
// we can directly import types from the NPM package
import {LfChecklistComponent, Checklist } from '@laserfiche/types-lf-ui-components';

@Component({
  selector: 'app-attachments',
  templateUrl:  './attachments.component.html',
  styleUrls: ['./attachments.component.css'],
})

export class ExampleApp implements AfterViewInit {

  // allows access to the LfChecklistElement in the html above, available starting in ngAfterViewInit hook
  @ViewChild('myChecklist') checklist!: ElementRef

  // type Checklist is the type of the custom element imported from the NPM package
  currentChecklist: Checklist[];

  // events for the elements are received as CustomEvents 
  onClick(event: CustomEvent <Checklist>	) {
    const checklist = event.detail;    // data will be stored in the detail property
  }

  ngAfterViewInit() {
    // the element's view is initialized after ngAfterViewInit
    this.checklist.nativeElement.initAsync(...)
  }
}

Using Via NPM

  1. It is more straightforward to use the NPM package.
  2. Install the npm package for the custom elements.

    npm install @laserfiche/lf-ui-components@NPM_VERSION

  3. Use the custom elements as defined in the documentation (lf-tree, etc.)
  4. There are other libraries available for use as well, that may be useful for you.
    • @laserfiche/lf-js-utils - this library contains utility functions that help avoid duplicate code.
    • @laserfiche/laserfiche-ui-components-services - this library contains implementations of the services used by the UI components. (Ex/ LfTreeService)
      1. These services connect with the Laserfiche Repository API to render real data from Laserfiche.
  5. Some components, specifically those with popups like dialogs, dropdowns, or menus, cannot be used inside jQuery modals due to our use of Angular Material. In order to use these components in a modal, we recommend using basic modals built with html/css/bootstrap.

Example: lf-checklist In an Angular Project Via NPM

In the example below, both the implementation and the typing are provided via @laserfiche/lf-ui-components/lf-checklist.

  1. npm install @laserfiche/lf-ui-components

  2. Inside ./app.module.ts, import the Angular modules. 

    import { NgModule } from '@angular/core';
import { LfChecklistModule } from '@laserfiche/lf-ui-components/lf-checklist';
@NgModule({
  declarations: [
     AppComponent,
     ...
  ],
  imports: [
     LfChecklistModule,
     BrowserAnimationsModule
  ]
 })

  3. Import the style sheet from the npm package by adding this line in styles.css in your angular application. 

    @import '~@laserfiche/lf-ui-components/cdn/lf-laserfiche-lite.css';
  4. The same usage of using lf-checklist in the html and ts file as used as CDN, except to import the component typing from @laserfiche/lf-ui-components 
    import { AfterViewInit, Component, ViewChild, ElementRef } from '@angular/core';
// we can directly import types from the NPM package
import { Checklist, LfChecklistComponent, LfChecklistService } from '@laserfiche/lf-ui-components/lf-checklist';

Setting UI Components Language

  1. The default language of the UI Components is the browser language.
  2. To explicitly set the language of the UI Components post a message as follows:
    3. const langObj = {
  'lf-localization-service-set-language': language
  }
window.postMessage(JSON.stringify(langObj), window.origin);
