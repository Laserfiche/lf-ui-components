Using Via CDN From jsdelivr

If you are using UI-components via CDN, we recommend getting them from jsdelivr because lfxstatic is not intended to serve the public and may not reliably deliver the files you want.

  1. Add the following script tags to index.html.

     <script src="https://cdn.jsdelivr.net/npm/zone.js@0.11.4/bundles/zone.umd.min.js"></script>
 <script src="https://cdn.jsdelivr.net/npm/@laserfiche/lf-ui-components@NPM_VERSION/cdn/lf-ui-components.js" defer></script>

    zone.js is a peer dependency of the UI components, and it needs to be loaded by the container application, before loading lf-ui-components.js.

    Note that Angular applications include zone.js by default in polyfills.ts. We recommend removing that import of zone.js, and loading it as shown above instead, to reduce the risks of loading zone.js twice.

  2. Choose a theme to style your components. Import the style sheet from the CDN by adding this line to the html file inside your application. You must choose either lf-laserfiche-lite or lf-ms-office-lite.

    <link href = "https://cdn.jsdelivr.net/npm/@laserfiche/lf-ui-components@NPM_VERSION/cdn/lf-laserfiche-lite.css" rel = "stylesheet" />

    or

    <link href = "https://cdn.jsdelivr.net/npm/@laserfiche/lf-ui-components@NPM_VERSION/cdn/lf-ms-office-lite.css" rel = "stylesheet" />

  3. If you are not using Angular Material, you must add an Angular Material stylesheet.

    <link href = "https://cdn.jsdelivr.net/npm/@laserfiche/lf-ui-components@NPM_VERSION/cdn/indigo-pink.css" rel = "stylesheet" />

  4. Add the types NPM package. This is not required but highly recommended if you are working in Typescript. Ensure the version matches the version of the script you added above.

    npm install @laserfiche/types-lf-ui-components@NPM_VERSION --save-dev

Using Via CDN From lfxstatic

If you wish to use lfxstatic, the instructions are the same, but the urls are different: 
  1.  <script src="https://lfxstatic.com/npm/-/zone.js@0.11.4/bundles/zone.umd.min.js"></script>
 <script src="https://lfxstatic.com/npm/@laserfiche/lf-ui-components@NPM_VERSION/cdn/lf-ui-components.js" defer></script>

  2. <link href = "https://lfxstatic.com/npm/@laserfiche/lf-ui-components@NPM_VERSION/cdn/lf-laserfiche-lite.css" rel = "stylesheet" />

    <link href = "https://lfxstatic.com/npm/@laserfiche/lf-ui-components@NPM_VERSION/cdn/lf-ms-office-lite.css" rel = "stylesheet" />

  3. <link href = "https://lfxstatic.com/npm/@laserfiche/lf-ui-components@NPM_VERSION/cdn/indigo-pink.css" rel = "stylesheet" />

Using Via NPM

  1. Install the UI components NPM package.

    npm install @laserfiche/lf-ui-components@NPM_VERSION

    This installs Angular components. You can use the strictly Angular components or the Angular elements by importing the relevant module into your app.module.ts.

    For example to use <lf-repository-browser> or <lf-repository-browser-component> you would do
    import {LfRepositoryBrowserModule} from '@laserfiche/lf-ui-components/lf-repository-browser';

  2. To use in a non-angular project:
    1. Either manually or as part of a build script, copy the CDN folder node_modules/@laserfiche/lf-ui-components/cdn/ into a destination served by your web server e.g.: static_assets
    2. Reference that location in a script tag within the HTML file in your project e.g.:

    <script src = "static_assets/lf-ui-components.js" defer ></script>

General Usage Tips

  1. UI components may be used with other Laserfiche libraries:
    • @laserfiche/lf-js-utils - this library contains utility functions and a reusable localization service.
    • @laserfiche/lf-ui-components-services - this library contains the data sources implementation to connect the ui components to live data from Laserfiche API services.
    • @laserfiche/lf-resource-library - this library contains localized strings resource files, fonts, and icons, that can be used to build Laserfiche Apps.
  2. Some components, specifically those with popups like dialogs, dropdowns, or menus, cannot be used inside jQuery modals due to our use of Angular Material. In order to use these components in a modal, we recommend using basic modals built with html/css/bootstrap.

Setting UI Components Language

  1. The default language of the UI Components is the browser language.
  2. To explicitly set the language of the UI Components post a message as follows:
    3. const langObj = {
  'lf-localization-service-set-language': language
}
window.postMessage(JSON.stringify(langObj), window.origin);
  3. To explicitly set debug mode for the UI Components post a message as follows:
    4.     const langObj = {
      'lf-localization-service-debug-mode': language
    }
    window.postMessage(JSON.stringify(langObj), window.origin);
