lf-folder-browser entrySelected:

Overview

Web Component to interact with data structures such as folders/files

Use Case

Selecting folder to save document to in Laserfiche

HTML Element

Attribute Value Description
description string displayed on top
multiple boolean allows multiple entries to be selected

Event Value Description
entrySelected/td> (event: CustomEvent<Entry[]>) => Promise<void> event raised when selectable node is selected

Method/Property Signature Description
initAsync (providers: LfRepositoryProvider, selectedEntryId?: string) => Promise<void> initializes the lf-repository-browser with a LfRepositoryProvider You must call this method. Each time this is called, the state of lf-repository-browser is reset. If selectedEntryId is specified, the tree is directly opened to the node with id = selectedEntryId.

LfRepositoryProvider

LfRepositoryService Interface

  • breadcrumbs : Entry[]
  • currentFolder : Entry | undefined
  • list : Entry[]
  • getRootEntryAsync() : Promise<Entry>
  • getData(folderId: string | null, filterText: string | undefined, refresh?: boolean): Promise<Entry[]>
    • getData should update the `list` property on the service
  • getParentEntryAsync(node: Entry): Promise<Entry | undefined>
    • If an entry does not have a parent (i.e. a root node), then we expect getParentEntryAsync to return Promise<undefined>
  • getNodeByIdAsync(id: string): Promise<TreeNode | undefined>
  • refreshAsync?(node: TreeNode): Promise<TreeNode[]>
    • Optional. If defined, users can refresh the tree. This function should not use cached values.

LfRepositoryProviders Interface

  • repositoryService: LfRepositoryService

Entry Interface

  • id : string
  • icon : string | string[]
  • isContainer : boolean
  • isLeaf : boolean
  • isSelectable: boolean
  • name : string
  • path : string
